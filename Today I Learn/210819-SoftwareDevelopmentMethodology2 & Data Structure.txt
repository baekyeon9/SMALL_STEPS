자료 구조 다루기 (Data Structure)

[ Stack 구현과 사용 ]

 - Last In Fi-rst Out (LIFO) 방식으로 데이터를 넣고 꺼낸다.
 - Data를 넣는 것을 'push'라고 하고, 데이터를 꺼내는 것을 'pop'이라고 한다.
 - 입력한 역순으로 데이터를 꺼내야하는 상황에서 이 자료구조를 사용한다.
 - 사용 예)
   1) 웹 브라우저에 방문 사이트의 URL을 보관할 때(이전 페이지)
   2) breadcrumb 구현 -stack 방식으로 자료를 구조화 하는 것이 적합하다.
      * 브레드크럼(빵부스러기로 표시한 길) : 헨젤과 그레텔에서 따온 용어로, 
        사이트나 웹 앱에서 유저의 위치를 보여주는 부차적인 네비게이션 시스템을 뜻한다.

소프트웨어 개발 방법론 (Software Development Methodology) 2

 [ Use-case 유스케이스 관계 : 포함 (Include) ]

   - 한 유스케이스에서 다른 유스케이스의 기능을 포함하는 관계
   - 도식화 : 기존 ·····> 다른 유스케이스 방향으로 점선 화살표
   - 예)

        회원가입         include       주소 검색
     (기본 유스케이스)  ···········>  (포함 유스케이스)
  

 [ Use-case 유스케이스 관계 : 확장 (Extend) ]

   - 어떤 조건에 의해 기존 유스케이스의 기능과 확장될 유스케이스와의 관계 
   - 도식화 : 확장될 유스케이스 ·····> 기존 방향으로 점선 화살표
   - 예)

        회원가입         extend        사진 등록
     (기본 유스케이스)  <···········  (확장 유스케이스)
